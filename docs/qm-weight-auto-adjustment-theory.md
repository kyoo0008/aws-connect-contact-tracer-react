# QM 평가 가중치 자동 산출 이론서

> **문서 목적**: QM 평가 결과 N건 누적 후, 카테고리(대항목) 및 서브아이템(소항목) 가중치를 통계 기반으로 자동 산출하는 방법론의 이론적 근거와 담당자 협의 사항을 정리한다.

---

## 1. 현황 및 문제 정의

### 1.1 현재 가중치 구조

```
QM 평가 폼
├── Category (대항목) × 8개
│   ├── weight: 10 (모두 균등)
│   └── SubItem (소항목) × 2~5개
│       └── weight: 1 (모두 균등, 일부 null)
```

| 항목 | 값 | 점수 반영 여부 |
|------|----|---------------|
| Category weight | 모두 10 | O — FAIL: -10, WARNING: -5 |
| SubItem weight | 모두 1 또는 null | X — 정의만 되어 있고 미사용 |

### 1.2 문제점

1. **변별력 부재**: 인사(greeting)와 정확성(accuracy)이 동일한 10점 배점으로, 상담 품질에 미치는 실제 영향도 차이가 반영되지 않음
2. **SubItem 가중치 사장**: 폼에 weight 필드가 존재하지만 점수 산정에 전혀 활용되지 않아, 카테고리 내 세부 항목 간 경중을 구분할 수 없음
3. **점수 분포 왜곡**: 균등 가중치로 인해 점수가 특정 대역에 몰리거나, 실질적으로 중요한 항목이 과소 반영될 수 있음

### 1.3 목표

> **평가 결과 데이터를 활용하여, 상담 품질을 가장 잘 변별하는 가중치 조합을 통계적으로 도출한다.**

## 2. 이론적 기반

### 2.1 문항 변별도 (Item Discrimination)

교육 측정학(Psychometrics)에서 시험 문항의 품질을 판단하는 핵심 지표이다.

#### 정의

> **변별도**: 특정 평가 항목이 "우수한 상담"과 "미흡한 상담"을 얼마나 잘 구분하는가

#### 계산 방법: Point-Biserial Correlation (점이연 상관계수)

각 평가 항목(subItem 또는 category)의 PASS/FAIL 여부와 최종 점수(finalScore) 간의 상관관계를 측정한다.

```
r_pb = (M_pass - M_fail) / S_total × √(p × q)

여기서:
  M_pass  = 해당 항목 PASS인 평가들의 finalScore 평균
  M_fail  = 해당 항목 FAIL인 평가들의 finalScore 평균
  S_total = 전체 finalScore의 표준편차
  p       = 해당 항목 PASS 비율
  q       = 1 - p (FAIL 비율)
```

**해석 기준:**

| r_pb 값 | 변별도 수준 | 가중치 방향 |
|---------|------------|------------|
| 0.40 이상 | 매우 우수 | 가중치 상향 |
| 0.30 ~ 0.39 | 우수 | 현행 유지 또는 소폭 상향 |
| 0.20 ~ 0.29 | 보통 | 현행 유지 |
| 0.20 미만 | 낮음 | 가중치 하향 검토 |

#### 예시

```
"공감 표현" subItem:
  - PASS인 상담들의 평균 finalScore = 82점
  - FAIL인 상담들의 평균 finalScore = 58점
  - 차이가 크다 → 변별도 높음 → 가중치 ↑

"외래어 및 전문 용어 사용" subItem:
  - 거의 모든 상담이 PASS (failRate ≈ 2%)
  - PASS/FAIL 그룹 간 finalScore 차이 미미
  - 변별도 낮음 → 가중치 ↓
```

### 2.2 정보 엔트로피 (Information Entropy)

정보 이론(Information Theory)에서 차용한 개념으로, 평가 항목의 "정보량"을 측정한다.

#### 정의

> **엔트로피**: 특정 항목의 결과가 얼마나 예측 불가능한가 (= 얼마나 다양한 결과를 생성하는가)

```
H(X) = -p × log₂(p) - (1-p) × log₂(1-p)

여기서:
  p = 해당 항목의 PASS 비율
```

**특성:**

```
p = 0.5 (반반)  → H = 1.0 (최대 엔트로피, 최대 정보량)
p = 0.9 (대부분 PASS) → H = 0.47 (낮은 정보량)
p = 1.0 (전부 PASS) → H = 0.0 (정보 없음)
p = 0.0 (전부 FAIL) → H = 0.0 (정보 없음)
```

#### 활용 의미

- **엔트로피가 높은 항목**: 상담원마다 결과가 다양 → 개인 차이를 반영 → 가중치 ↑
- **엔트로피가 낮은 항목**: 거의 모든 상담원이 같은 결과 → 변별 기여도 낮음 → 가중치 ↓

### 2.3 종합 가중치 산출 공식

변별도와 엔트로피를 결합하여 최종 가중치를 산출한다.

```
RawWeight(i) = |r_pb(i)| × H(i)

여기서:
  r_pb(i) = 항목 i의 점이연 상관계수 (변별도)
  H(i)    = 항목 i의 정보 엔트로피

최종 가중치:
  Weight(i) = clamp(normalize(RawWeight(i)) × TotalWeight, MinWeight, MaxWeight)

제약 조건:
  Σ Weight(i) = TotalWeight (예: 100)
  MinWeight ≤ Weight(i) ≤ MaxWeight
```

**왜 곱인가?**

| 변별도 | 엔트로피 | 종합 | 해석 |
|--------|---------|------|------|
| 높음 | 높음 | **높음** | 결과가 다양하면서 품질과 강하게 연관 → 가장 유용한 항목 |
| 높음 | 낮음 | 보통 | 품질과 연관되지만 결과가 편향 → 유용하나 제한적 |
| 낮음 | 높음 | 보통 | 결과는 다양하지만 품질과 무관 → 노이즈에 가까움 |
| 낮음 | 낮음 | **낮음** | 변별력도 없고 정보도 없음 → 가중치 최소화 |

## 3. 적용 레벨별 설계

### 3.1 Category(대항목) 가중치 자동 산출

**입력 데이터:**

```
N건의 평가 결과에서:
  - categoryResult: PASS / WARNING / FAIL (수치화: 1.0 / 0.5 / 0.0)
  - finalScore: 0~100
```

**산출 절차:**

```
Step 1. 각 카테고리별 점이연 상관계수 계산
        X = [카테고리 결과 수치] (0.0, 0.5, 1.0)
        Y = [finalScore]
        r_pb = corr(X, Y)

Step 2. 각 카테고리별 결과 분포 엔트로피 계산
        3-class 엔트로피:
        H = -p_pass×log₂(p_pass) - p_warn×log₂(p_warn) - p_fail×log₂(p_fail)

Step 3. 종합 가중치 산출
        RawWeight = |r_pb| × H

Step 4. 정규화 및 제약 적용
        TotalWeight = 100, MinWeight = 5, MaxWeight = 30
```

**예상 결과 (가상):**

| 카테고리 | 현재 weight | 변별도 | 엔트로피 | 산출 weight |
|----------|-----------|--------|---------|-------------|
| accuracy | 10 | 0.45 | 0.72 | **18** |
| efficiency | 10 | 0.38 | 0.85 | **16** |
| proactivitiness | 10 | 0.42 | 0.61 | **14** |
| voiceProduction | 10 | 0.25 | 0.90 | **12** |
| greeting | 10 | 0.20 | 0.78 | **10** |
| speed | 10 | 0.22 | 0.65 | **10** |
| waitManagement | 10 | 0.15 | 0.55 | **10** |
| languageUse | 10 | 0.10 | 0.30 | **10** |
| **합계** | **80** | | | **100** |

> 정확성, 효율성처럼 상담 품질과 높은 상관관계를 보이는 항목의 배점이 올라가고,
> 언어사용처럼 대부분 PASS인 항목은 최소 가중치로 수렴한다.

### 3.2 SubItem(소항목) 가중치 자동 산출

**전제 조건:** 평가 결과에 `subItemId`가 포함되어야 함 (output format 개선 필요)

**입력 데이터:**

```
N건의 평가 결과에서 (카테고리별):
  - criteria[].subItemId: 소항목 식별자
  - criteria[].status: PASS / FAIL
  - 해당 카테고리의 categoryResult: PASS / WARNING / FAIL
```

**산출 절차:**

```
Step 1. 카테고리 내 각 subItem별 점이연 상관계수 계산
        X = [subItem PASS=1, FAIL=0]
        Y = [해당 카테고리 결과 수치] (PASS=1.0, WARNING=0.5, FAIL=0.0)
        r_pb = corr(X, Y)

Step 2. 각 subItem별 이진 엔트로피 계산
        H = -p×log₂(p) - (1-p)×log₂(1-p)
        (p = PASS 비율)

Step 3. 종합 가중치 산출 및 카테고리 내 정규화
        RawWeight = |r_pb| × H
        Weight = RawWeight / Σ(RawWeight)  → 합계 1.0
```

## 4. 안전장치

### 4.1 지수이동평균 (EMA) 스무딩

가중치가 한 번의 배치 계산으로 급격히 변하지 않도록 이전 값과 혼합한다.

```
AppliedWeight = α × CalculatedWeight + (1 - α) × CurrentWeight

권장값: α = 0.3 (약 70%는 기존 가중치 유지)
```

**스무딩 효과 예시:**

```
현재 weight = 10
계산된 weight = 18

1차 적용: 0.3 × 18 + 0.7 × 10 = 12.4
2차 적용: 0.3 × 18 + 0.7 × 12.4 = 14.1
3차 적용: 0.3 × 18 + 0.7 × 14.1 = 15.3
...
수렴: → 18
```

약 5~7회 조정 사이클 후 목표값에 수렴한다. 이상치(outlier) 데이터가 유입되어도 영향이 완화된다.

### 4.2 최소/최대 가중치 제한

```
MinWeight = 5   → 어떤 카테고리도 완전히 무시되지 않음
MaxWeight = 30  → 단일 카테고리가 과도하게 지배하지 않음
TotalWeight = 100
```

### 4.3 최소 데이터 건수 요구

```
MinEvaluationCount = 50건 (권장)

50건 미만: 자동 산출 비활성, 수동 가중치 유지
50건 이상: 통계 산출 시작, 참고용 리포트 제공
100건 이상: 자동/반자동 적용 가능
```

**근거:** 점이연 상관계수의 안정적 추정을 위해 최소 30건 이상의 표본이 필요하며, 3-class(PASS/WARNING/FAIL) 분포의 경우 50건 이상이 적절하다.

### 4.4 이상 탐지 (Anomaly Detection)

가중치 변동이 비정상적으로 클 경우 자동 적용을 중단한다.

```
변동률 = |NewWeight - CurrentWeight| / CurrentWeight × 100

변동률 > 50%: 경고 알림, 관리자 확인 후 적용
변동률 > 100%: 자동 적용 차단, 수동 검토 필수
```

## 5. 실행 계획

### 5.1 데이터 파이프라인

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ QM 평가 수행  │────▶│ 결과 저장 (DDB)   │────▶│ 통계 집계 배치    │
│ (Gemini)     │     │ + subItemId 포함  │     │ (주 1회 or N건)  │
└──────────────┘     └──────────────────┘     └────────┬─────────┘
                                                       │
                     ┌──────────────────┐     ┌────────▼─────────┐
                     │ 가중치 적용       │◀────│ 가중치 산출       │
                     │ (Form 업데이트)   │     │ (변별도+엔트로피) │
                     └──────────────────┘     └──────────────────┘
```

### 5.2 단계별 일정

| 단계 | 작업 | 선행 조건 |
|------|------|----------|
| **A** | 평가 결과에 `subItemId` 포함 | output format 수정 |
| **B** | subItem별 통계 집계 인프라 구축 | A 완료 + 데이터 누적 시작 |
| **C** | 변별도/엔트로피 계산 모듈 개발 | B 완료 |
| **D** | 가중치 산출 리포트 생성 (참고용) | C 완료 + 50건 이상 누적 |
| **E** | 관리자 승인 기반 반자동 적용 | D 완료 + 담당자 합의 |
| **F** | (선택) 완전 자동 적용 | E 완료 + 충분한 운영 검증 |

## 6. 담당자 협의 사항

### 6.1 사업/기획 담당자 협의

| # | 협의 항목 | 선택지 | 권장 | 비고 |
|---|----------|--------|------|------|
| 1 | **가중치 조정 주기** | 주 1회 / 격주 / 월 1회 / N건 누적 시 | 월 1회 또는 100건 누적 시 | 너무 잦으면 평가 기준 불안정, 너무 드물면 반영 지연 |
| 2 | **적용 방식** | 완전 자동 / 관리자 승인 후 적용 / 참고용 리포트만 | 관리자 승인 후 적용 | 초기 운영 시 수동 검증 필수 |
| 3 | **최소 데이터 건수** | 30건 / 50건 / 100건 | 50건 | 통계적 유의성과 운영 현실 사이 균형 |
| 4 | **가중치 총합** | 80점(현행) / 100점 | 100점 | 100점 기준이 직관적이며 배점 해석 용이 |
| 5 | **Min/Max 가중치** | Min 5~10 / Max 20~30 | Min 5, Max 30 | Max를 너무 높이면 단일 항목이 결과를 지배 |
| 6 | **EMA 스무딩 계수 (α)** | 0.2 / 0.3 / 0.5 | 0.3 | 낮을수록 보수적(변화 느림), 높을수록 민감 |
| 7 | **기존 평가 소급 적용 여부** | 소급 재계산 / 신규 건만 적용 | 신규 건만 적용 | 소급 시 이미 확정된 평가 점수 변동 이슈 |
| 8 | **카테고리 최소 가중치 보장 정책** | 변별도 무관 최소 보장 / 0까지 허용 | 최소 보장 (Min 5) | 고객 경험상 중요한 항목이 통계적으로만 저평가되는 것 방지 |

### 6.2 QA/QM 담당자 협의

| # | 협의 항목 | 논의 포인트 |
|---|----------|------------|
| 1 | **SubItem ↔ Criteria 매핑 정합성** | 현재 form의 subItem 수(2~5개)와 실제 평가 결과의 criteria 수(4~10개)가 불일치. subItemId를 결과에 포함시키는 방식으로 해결할지, 아니면 form의 subItem을 criteria 단위로 세분화할지 결정 필요 |
| 2 | **카테고리 판정 기준 변경** | 현재 Gemini 프롬프트가 자체적으로 PASS/WARNING/FAIL을 판정. SubItem 가중치 기반 정량적 판정(가중 달성률 threshold)으로 전환할지 협의 |
| 3 | **가중치 변경 시 통보 절차** | 가중치가 바뀌면 상담사의 체감 평가 난이도가 달라짐. 변경 전후 영향도 분석 및 공지 프로세스 필요 |
| 4 | **도메인 전문가 오버라이드** | 통계적으로 낮은 변별도를 보이더라도 QM 정책상 반드시 높은 가중치를 유지해야 하는 항목이 있는지 확인 (예: 정확성은 법적 이슈와 연관) |
| 5 | **평가 기간/시즌 효과** | 성수기/비수기, 신입/경력 상담사 비율 변화에 따른 통계 왜곡 가능성. 기간 필터링 또는 세그먼트별 가중치 필요 여부 |

### 6.3 개발 담당자 협의

| # | 협의 항목 | 논의 포인트 |
|---|----------|------------|
| 1 | **output format 변경 범위** | `formatOutputFormat()`에 `subItemId` 추가 시 기존 결과 파싱 로직, 프론트엔드 호환성 영향 범위 확인 |
| 2 | **통계 집계 저장소** | 기존 DynamoDB 테이블에 통계 항목 추가 vs 별도 집계 테이블 생성 |
| 3 | **배치 실행 인프라** | Lambda scheduled event / Step Functions / EventBridge 중 선택 |
| 4 | **Form 버전 관리** | 가중치 변경 시 form version bump 정책 — 기존 진행 중 평가에 영향 없도록 격리 |
| 5 | **롤백 전략** | 가중치 적용 후 문제 발견 시 이전 가중치로 즉시 복원할 수 있는 메커니즘 |

### 6.4 반드시 합의해야 하는 핵심 결정 사항

```
┌─────────────────────────────────────────────────────────────────┐
│                    핵심 합의 필요 사항 (3가지)                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ❶ 자동 vs 반자동                                               │
│     "산출된 가중치를 자동 적용할 것인가,                          │
│      관리자 승인을 거칠 것인가?"                                  │
│     → 초기에는 반자동(리포트 + 승인) 권장                        │

│      어느 쪽을 우선할 것인가?"                                    │
│     → 정책 기반 floor/ceiling 설정으로 해결 가능                 │
│                                                                  │
│  ❸ 소급 적용 여부                                                │
│     "새 가중치로 과거 평가를 재계산할 것인가?"                    │
│     → 확정된 평가의 안정성을 위해 신규 건만 적용 권장             │
```

## 7. 기대 효과

### 7.1 정량적 효과

| 지표 | 현재 (균등 가중치) | 개선 후 (자동 산출) |
|------|-------------------|-------------------|
| 점수 분포 표준편차 | 낮음 (점수 밀집) | 적정 수준으로 확대 |
| 카테고리 간 변별 기여도 편차 | 없음 (모두 동일) | 실제 영향도 반영 |
| 상-하위 상담사 점수 차이 | 작음 | 확대 (변별력 향상) |
| 평가 결과의 예측 타당도 | 미검증 | 측정 및 추적 가능 |

### 7.2 정성적 효과

- **공정성 향상**: 실제 상담 품질과 높은 상관관계를 보이는 항목에 더 높은 배점을 부여
- **코칭 정밀도**: 가중치가 높은 항목에서의 개선이 점수에 더 크게 반영되어, 상담사 코칭 우선순위가 명확해짐
- **평가 기준 투명성**: 가중치 산출 근거(변별도, 엔트로피)가 데이터로 설명 가능

## 8. 리스크 및 대응

| 리스크 | 영향 | 대응 방안 |
|--------|------|----------|
| 표본 편향 (특정 상담 유형 과대 대표) | 가중치 왜곡 | 상담 유형별 층화 샘플링, 기간 필터 |
| 순환 논리 (가중치가 점수에 영향 → 점수로 가중치 재산출) | 가중치 수렴 불안정 | EMA 스무딩, 변동률 상한, 충분한 관찰 기간 |
| Gemini 모델 변경 시 평가 패턴 변화 | 기존 통계 무효화 | 모델 변경 시 통계 리셋 또는 별도 기간 집계 |
| 상담사 반발 (갑작스러운 점수 변동) | 운영 이슈 | 단계적 적용 (α=0.3), 사전 공지, 시뮬레이션 결과 공유 |
| 소항목 간 다중공선성 (Multicollinearity) | 가중치 불안정 | VIF 검사, 상관관계 높은 항목 그룹핑 검토 |

- **점이연 상관계수**: 이진 변수와 연속 변수 간 상관관계 측정 방법
- **지수이동평균 (EMA)**: 시계열 데이터의 스무딩 기법
- **교육 측정학 (Psychometrics)**: 시험 문항 분석 및 배점 최적화 이론
